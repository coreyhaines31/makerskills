# Content types

Three types. Weekly target: 2 promo + 2–3 BIP + 2–3 educational = 5–7 posts.

---

## promo (~2/week, one of the 6 properties)

Direct mention of one property by name. **Default: no link in the body.** If a link is included, it goes in a first comment below the post (Typefully supports this on LinkedIn) or a reply on X — never inline. See `voice.md`.

**Hook = the angle, not "check out X."** Best openers:

- Specific outcome: *"I just shipped a CRO audit that cut signup friction by 38%."*
- Contrarian take: *"Most CRO playbooks tell you to A/B test. Most teams shouldn't."*
- Story open: *"Two weeks ago, a SaaS founder told me their bounce rate was 'fine.' It was 31%."*
- One specific lesson: *"The most underrated chapter in Mental Models For Marketing: 'Loss aversion ≠ scarcity.'"*

**Structure**:
```
[hook — a take, outcome, or story]
[1–2 sentence development naming the property]
[conviction-coded close — invites action without linking]
```

If a first-comment link is being added, draft it separately as:
```
[short context line + URL]
```

**Decision: link or no link?**
- No link: when the post stands alone — readers can find the property by name. Default for short promo, BIP-adjacent promo, and when the property is well-known to the audience.
- Link in first comment: when there's a specific destination (an ebook, a new feature page, a newsletter issue) that wouldn't be obvious from the property name alone.

**Avoid**:
- "Just dropped X, link below"
- "Excited to share..."
- Anything that sounds like a press release

## build-in-public / BIP (~2–3/week)

Behind-the-scenes of running CF, Swipe Files, Magister, Truelist, or Corey's day. Usually no CTA, no URL.

**Best openers**:
- A number: *"$X MRR this month."* / *"Took 14 sales calls this week."*
- A screenshot description: *"Here's what our pipeline looked like Monday morning."*
- A lesson: *"Here's what didn't work this quarter."*
- A workflow: *"How I'm using Claude Code + cf-skills to draft client decks."*

**Structure**:
```
[concrete fact, number, or moment]
[the lesson or surprise]
[optional: question or observation that invites reply]
```

**Topics that work**:
- Revenue / pipeline updates
- What Corey shipped this week
- A tool / process change
- A mistake + what he learned
- Behind-the-scenes of a project

## educational (~2–3/week, no plug)

Standalone insight or framework. No property URL. No CTA. Just teach one thing.

**Best openers**:
- A framework: *"Three signals you're charging too little:"*
- A counterintuitive observation: *"Email open rates went up when we stopped writing subject lines for clicks."*
- A pattern: *"Every B2B SaaS pricing page makes the same mistake on the enterprise tier."*

**Structure**:
```
[take or observation]
[the explanation in 1–3 sentences, or a short list]
[optional: the implication]
```

**Topics that work**:
- CRO / pricing / positioning principles
- Marketing-x-AI trends and predictions
- Tactical observations from client work (anonymized)
- A reframe of a common piece of advice

## Cross-type mixing rules

- Two consecutive promo posts → don't. Space them.
- Two consecutive BIP posts → fine if different topics.
- A promo post the day after a BIP that mentioned the same property → cannibalization, skip or change angle.
- Educational that *implicitly* showcases a property's value is great, but don't link to the property — that turns it into promo.
